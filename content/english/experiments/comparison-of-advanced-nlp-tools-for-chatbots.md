<a class="twitter-share-button" data-count="horizontal" data-text="Comparison of advanced NLP tools for chatbots" data-url="http://experiencesutra.com/experiments/comparison-of-advanced-nlp-tools-for-chatbots/" href="http://twitter.com/share"></a>

</div><div style="clear:both"></div><div style="padding-bottom:4px;"></div><blockquote><p>&nbsp;“Chatbots represent a new trend in how people access information, make decisions, and communicate. We think that chatbots are the beginning of a new form of digital access, which centers on messaging. Messaging has become a huge component of how we interact with our devices, and how we stay connected with the people, businesses and the day-to-day activities of life. Chatbots bring commerce into this part of our lives, and will open up new opportunities.” &nbsp;– <a href="https://www.linkedin.com/in/christiepitts" target="_blank">Christie Pitts</a>, Manager, Verizon Ventures</p></blockquote>
<p>We explored various NLP engine for chatbots and here is the comparison of prominent NLP tools based on our experience/understanding.</p>
<table>
<tbody>
<tr>
<td width="120"><strong>Feature</strong></td>
<td width="133"><a href="https://wit.ai/home">Facebook wit.ai</a></td>
<td width="128"><a href="https://www.luis.ai/">Microsoft luis.ai</a></td>
<td width="114"><a href="https://www.ibm.com/watson/developercloud/nl-classifier.html">IBM Watson language classifier</a></td>
<td width="128"><a href="https://api.ai/">api.ai</a></td>
</tr>
<tr>
<td width="120"><strong>Price</strong></td>
<td width="133">Free/no limitations</td>
<td width="128">Free (in beta) up to 10k transactions per month, 5 requestsper second, per account</td>
<td width="114">First classifier is FREE. Each additional classifier is $20.00 per month.First one thousand API calls per month are FREE. Additional calls are $0.0035 per call.First four training events per month are FREE. Additional training events are $3.00 per training event.</td>
<td width="128">Free up to 6,000 requests,Enterprise starts at $89 USD per month</td>
</tr>
<tr>
<td width="120"><strong>Ease of training</strong></td>
<td width="133">Web based interface. Could not find option to import already trained model</td>
<td width="128">Easy to use web based interface. Ability to import utterances.</td>
<td width="114">Need to provide the data to train the Natural Language Classifier in comma-separated value (CSV) format specified on the website.</td>
<td width="128">Easy to use web based interface. Ability to import intents and entities.</td>
</tr>
<tr>
<td width="120"><strong>Pre-built features/entity</strong></td>
<td width="133">It has pre-built entities like datetime, duration, location, number, amount_of_money, phone number, url, email,</td>
<td width="128">It has pre-built entities like number(in number or text), temperature, dimensions, money, age, geography, encyclopedia, percentage and datetime</td>
<td width="114">There is no single API that does intent and entity recognition in a single call, this is inconvenient since you have to build your own pipeline with multiple API calls to extract all the information you need and it’s harder to link entities to intent</td>
<td width="128">built-in domains of knowledge (Intents with Entities and even suggested Replies ) on topics like small talk, weather, apps or even wisdom.</td>
</tr>
<tr>
<td width="120"><strong>Integration effort</strong></td>
<td width="133">No integration module, webservice API</td>
<td width="128">Integration into MS Azure andother services, deployable in anysupported servers</td>
<td width="114">There are a lot of building blocks that you can use in your application, but you probably will spend a decent amount of time integrating them into one solution.</td>
<td width="128">Integration module to connect to messenger APIs. Support fordeploying into Heroku server,enterprise paid environment</td>
</tr>
<tr>
<td width="120"><strong>API</strong></td>
<td width="133">Wit.ai API for iOS, Android,Nod.js, Raspberry Pi, Ruby,Python, C, Rust and Windowsphone</td>
<td width="128">No coding needed</td>
<td width="114">For simple scripted conversation, IBM provides the Dialog API. Dialog API is not integrated with text understanding APIs and you may need to write thousands of line of XML to build a simple app</td>
<td width="128">SDKs for Android, iOS, AppleWatch, Node.js, Cordova, Unity,C#, Xamarin, Windows phone,Python and JS</td>
</tr>
<tr>
<td width="120"><strong>Content addition and updating</strong></td>
<td width="133">Web based interface to updated/add new entity/chat flow</td>
<td width="128">Web based interface to update/delete/add intent/entity</td>
<td width="114">There is no single API that does intent and entity recognition in a single call.</td>
<td width="128">Web based interface to update/delete/add intent/entity</td>
</tr>
<tr>
<td width="120"><strong>Notes</strong></td>
<td width="133">Wit.ai supports 50 different languages including English, Chinese, Japanese, Polish, Ukrainian and RussianProvides a nice combination of both voice recognition and machine learning for developers.</td>
<td width="128"></td>
<td width="114">The classifier supports English (en), Arabic (ar), French (fr), German (de), Japanese (ja), Italian (it), Portuguese (pt), and Spanish (es).</td>
<td width="128">Speech to Text and Text to Speech capabilities, along with machine&nbsp; learning.</td>
</tr>
</tbody>
</table>
